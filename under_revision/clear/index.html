rsync

Должна быть установлена на всех машинах.

В rsync всегда, сначала указываем источник, а потом место назначения, т.е сначала что копируем, а потом куда копируем. Всегда используйте параметр -nv перед прогоном. rsync может натворить огромную головную боль если что-то не так сделать.

Завершающий слеш "/"

Источник без завершающего слеша скопирует сам каталог /path/dir в удаленный каталог /dest_dir: ]$ rsync -a /path/dir user@host:dest_dir

Путь к каталогу источнику с косой чертой в конце скопирует не сам каталог /path/dir, а все содержимое в удаленный каталог /dest_dir:]$ rsync -a /path/dir/ user@host:dest_dir

Небольшая инструкция по синхронизации

  1. Синхронизируем локальный каталог с удаленным сервером: ]$ rsync -a -e "ssh -p 7967" html/ user@host:html/

    Примечание: -e "ssh -p 7967" указывает коннектиться по определенному порту.

  2. На удаленной машине синхронизируем каталоги: ]$ rsync -a html/ /var/www/html/

Исключить каталог из операции переноса

]$ rsync -a -e "ssh -p 7967" --exclude 'res/header' html/ user@host:html/

Команда scp

scp - копирует файлы между хостами в Сети. Работает подобно команде cp

Примеры:

  • Файл с удаленного хоста к себе в текущую директорию - ]$ scp user@host:path_file path_copy_past

  • Каталог с удаленного хоста через другой порт - ]$ scp -r -P 2871 user@host:path_file path_copy_past

Внимание: для scp порт указывается с большой "P".

Инструкция по настройке сервера

Инструкция универсальна и адаптирована для семейств RHEL/CentOS/AlmaLinux и Debian/Ubuntu.

1. Настройка записей DNS

Нужно в запись А добавить ip-адрес сервера. Для www.you_name_domain.ru и для you_name_domain.ru

1. Создаем пользователя с правами sudo

  • ]# useradd new_username - добавляем пользователя. (для Debian adduser)
  • ]# passwd new_username - задаем ему пароль.
  • ]# usermod -aGgroup_name1 group_name2... user_name - добавляем пользователя в группу wheel. (В Debian: установить apt install sudo. Добавить в группу sudo)

Выходим из сессии root и заходим под новым пользователем.

2. Настройка SSH

Внимание. Для генерации ключей-SSH в Windows 11, смотрите: Генерация ключей-ssh в терминале Windows

2.1 Генерация ключа и копирование открытого ключа на сервер

  • ]$ ssh-keygen — создаём пару ключей на станции.
  • ]$ ssh-copy-id user@host — отправляем публичный ключ на удалённый сервер.

2.2 Настройка файла /etc/ssh/sshd_config

  • Отключить авторизацию по паролю — PasswordAuthentication no
  • Запретить вход по root — PermitRootLogin no
  • Сменить порт — Port #порт

Для того чтобы изменения вступили в силу, необходимо перезагрузить SSH-сервер. Команда перезагрузки приведена в пункте 2.5 — её следует выполнять только после завершения всех предыдущих настроек. Если применить перезагрузку до открытия нового порта в firewall и MAC, вы потеряете доступ к серверу.

2.3 Разрешить системе обязательного доступа (SELinux / AppArmor) использовать нестандартный порт

SELinux (CentOS/RHEL) и AppArmor (Debian/Ubuntu) — системы обязательного контроля доступа (MAC — Mandatory Access Control). Они ограничивают процессы, в том числе могут блокировать использование нестандартных портов.

2.4 Открытие порта в межсетевом экране (firewall)

2.5 Применение конфигурации и проверка

  • Перезагрузить конфигурацию SSH-сервера ] systemctl reload sshd
  • Подключиться по новому порту ] ssh -p #порт user@host

Не закрывайте текущую SSH-сессию, пока не убедитесь, что подключение по новому порту работает. Если что-то пойдёт не так, вы сможете исправить ошибку в открытой сессии.
