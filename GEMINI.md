# Обзор проекта: linuxgik.ru

Персональный справочный портал и блог, посвященный информационным технологиям, администрированию Linux и веб-разработке. Проект построен на базе генератора статических сайтов **Jekyll**.

## Основные технологии
- **Генератор статических сайтов:** [Jekyll](https://jekyllrb.com/)
- **Стилизация:** Чистый CSS с использованием CSS Grid и кастомных переменных (`assets/css/`).
- **Шаблонизация:** Liquid (используется в `_layouts`, `_includes` и `_posts`).
- **Управление данными:** YAML (используется в `_data/navigation.yml` и фронтматере страниц).

## Архитектура и структура

### Ключевые директории
- `_includes/`: Содержит переиспользуемые фрагменты HTML.
  - `logika_header_sidebar.html`: Основная логика определения текущего раздела, подраздела и соответствующего сайдбара на основе URL.
  - `header_nav.html`: Отрисовка многоуровневой навигации и хлебных крошек.
- `_layouts/`: Определяет структуру страниц.
  - `default.html`: Базовый шаблон для всех страниц.
  - `post.html`: Шаблон для записей блога.
- `_data/`: Конфигурационные данные и навигация.
  - `navigation.yml`: Описывает иерархию сайта, включая разделы, подразделы и соответствующие ссылки/сайдбары.
- `_posts/`: Markdown-файлы для записей блога (например, `2026-04-05-hello.md`).
- `assets/`: Статические ресурсы, включая CSS, JS, изображения и шрифты.
- `pages/`, `web-developers/`: Основные директории с контентом и организованными статьями.

### Логика навигации
Сайт использует динамическую систему навигации, управляемую через `_data/navigation.yml`. Логика в `_includes/logika_header_sidebar.html` сопоставляет текущий `page.url` с данными навигации для установки переменных `current_section` и `current_subsection`, которые затем используются в `header_nav.html` для выделения активных ссылок и генерации хлебных крошек.

## Сборка и запуск

### Предварительные требования
- Ruby
- Bundler

### Команды
- **Установка зависимостей:**
  ```bash
  bundle install
  ```
- **Локальный запуск (разработка):**
  ```bash
  bundle exec jekyll serve
  ```
- **Сборка сайта:**
  ```bash
  bundle exec jekyll build
  ```

## Конвенции разработки

- **Создание контента:** Новые статьи должны размещаться в соответствующей поддиректории (например, `pages/` или `_posts/`) с корректным фронтматером Jekyll (layout, title и т.д.).
- **Обновление навигации:** Чтобы добавить новый раздел или страницу в навигацию, обновите `_data/navigation.yml`.
- **Стилизация:** Следуйте существующей системе CSS-переменных, определенных в `assets/css/variables/` и `assets/css/color-var.css`. Используйте сеточную разметку из `assets/css/grid-layout.css`.
- **Именование:** Используйте нижний регистр и дефисы для имен файлов и URL-адресов.
